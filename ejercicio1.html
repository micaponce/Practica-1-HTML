Practica 1 - HTML

EJERCITACIÓN

1. ¿Qué es HTML, cuando fue creado, cuáles fueron las distintas versiones y cuál es la última?

HTML son las siglas en inglés de HyperText Markup Language (Lenguaje de Marcas de Hipertexto). Es el lenguaje de marcado principal utilizado para la elaboración de páginas web y actúa como un estándar que define la estructura básica y el código necesario para presentar contenido como texto, imágenes, videos y juegos. No se considera un lenguaje de programación, ya que no posee funciones aritméticas ni variables; su función es puramente descriptiva mediante el uso de etiquetas

Fue desarrollado originalmente por Tim Berners-Lee a principios de la década de los 90 y se popularizó masivamente gracias al navegador Mosaic en 1993.

  • HTML (1990-1995): Durante sus primeros cinco años, el lenguaje pasó por una serie de revisiones y extensiones alojadas inicialmente en el CERN y luego en el IETF. Su propósito original era describir semánticamente documentos científicos
  • HTML 3.0 (1995): Fue un primer intento del W3C por extender el lenguaje, pero resultó frustrado y no llegó a consolidarse como estándar
  • HTML 3.2 (1997): Tras el fracaso de la versión 3.0, se adoptó un enfoque más pragmático que se completó en 1997, permitiendo una adopción más uniforme entre los navegadores de la época
  • HTML 4.01 (1997): Se publicó poco después de la versión 3.2 en el mismo año. Es una versión de alto rigor técnico que introdujo las Declaraciones de Tipo de Documento (DTD) en variantes Transitoria (más flexible) y Estricta
  • XHTML 1.0 (2000): Consistió en una reformulación de HTML 4.01 bajo las reglas de XML. No agregó funciones nuevas, sino que se centró en una nueva forma de serialización del código para hacerlo más estructurado
  • XHTML 2.0 (Posterior a 2000): Fue un proyecto del W3C que buscaba un lenguaje nuevo que no era compatible con las versiones anteriores de HTML y XHTML. Debido a este conflicto con la dirección de la web, la propuesta fue finalmente rechazada en 2004
  • HTML 5 (2012 - Actualidad): Es la quinta y última revisión importante del lenguaje. Aunque su desarrollo comenzó fuera del W3C en 2004 (por el grupo WHATWG), el W3C se unió al proyecto en 2007. En 2012 se presentó el equipo para crear la Recomendación oficial HTML5 es la versión estándar actual adoptada por todos los navegadores modernos (Chrome, Firefox, Safari, Opera). Sus principales innovaciones incluyen:
    • Multimedia Nativa: Permite añadir audio y video con las etiquetas < audio > y < video > sin necesidad de plugins externos como Flash
    • Gráficos y Animaciones: Introduce la etiqueta < canvas > para crear dibujos y animaciones 2D mediante JavaScript
    • Semántica mejorada: Incorpora etiquetas como < header >, < nav >, < section > y < footer > que ayudan a organizar el código de forma más clara y descriptiva
    • Formularios avanzados: Añade validaciones automáticas (atributo required), tipos de entrada específicos (email, url, date, range) y foco automático (autofocus)
    • APIs y Almacenamiento: Incluye capacidades de geolocalización y Web Storage (localStorage), permitiendo guardar datos en el navegador de forma más segura que con las antiguas cookies
    • Diseño Adaptativo: Es totalmente compatible con dispositivos móviles, permitiendo que las páginas se adapten a diferentes pantallas
    • Actualmente, mientras el W3C publica recomendaciones numeradas, el grupo WHATWG mantiene HTML como un estándar vivo (Living Standard) en constante actualización

2. ¿Cuáles son los principios básicos que el W3C recomienda seguir para la creación de documentos con HTML?

  • Interoperabilidad: El objetivo principal es que los documentos HTML funcionen correctamente en diferentes navegadores y plataformas. Al seguir estándares comunes, se evita tener que desarrollar múltiples versiones de un mismo documento, reduciendo costos y esfuerzos
  • Independencia del dispositivo: Los documentos deben crearse bajo la premisa de que cualquier tipo de dispositivo (PCs, teléfonos móviles, dispositivos de mano o sistemas de voz) debe ser capaz de acceder y usar la información de la Web
  • Separación de Contenido y Presentación: El W3C recomienda que el HTML se encargue únicamente de la estructura y el contenido, mientras que las Hojas de Estilo (CSS) asuman la responsabilidad de la presentación (fuentes, colores, alineación). Por esta razón, el W3C ha ido "desaprobando" etiquetas y atributos que solo servían para dar formato visual
  • Uso de Convenciones Estándar: La Web depende de que tanto autores como empresas compartan las mismas convenciones y especificaciones del lenguaje para garantizar un crecimiento ordenado a largo plazo

3. En las Especificaciones de HTML, ¿cuándo un elemento o atributo se considera desaprobado? ¿y obsoleto?

  • Desaprobado: Un elemento que se desaconseja usar porque existen mejores formas de lograr el mismo efecto (usualmente mediante CSS), pero que aún es soportado por compatibilidad.
  • Obsoleto: Elementos que ya no forman parte de la especificación oficial y pueden no ser interpretados por los navegadores modernos.

4. Qué es el DTD y cuáles son los posibles DTDs contemplados en la especificación de HTML 4.01?

El DTD (Document Type Declaration o Declaración del Tipo de Documento) es una instrucción que se ubica en la primera línea de un archivo HTML, apareciendo antes de la etiqueta de apertura < html >. Su función principal es indicar al navegador bajo qué especificación o estándar de HTML está construido el documento para que este pueda interpretarlo correctamente.

5. ¿Qué son los metadatos y cómo se especifican en HTML?

Son datos que describen información sobre el documento (como palabras clave o expiración) que no se muestran al usuario. Se especifican con la etiqueta < meta > dentro de la sección < head > < / head >
